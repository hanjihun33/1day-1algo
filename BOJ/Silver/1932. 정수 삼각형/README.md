# BOJ 1932 정수 삼각형
### [문제 바로가기](https://www.acmicpc.net/problem/1932)
| 난이도 | 정답률 | 분류 |
| ------ | ------ | ------ |
|  Silver I | 60.430% | dp |


### 구현
---
아래로 내려갈 때 각 칸에서는 대각선 왼쪽/오른쪽으로만 이동 가능하다.
따라서 각 칸에 도착했을 때 만들 수 있는 최대 누적 합을 dp로 저장하면, 마지막 줄의 최댓값이 정답이 된다.

맨 왼쪽, 맨 오른쪽의 경우 위에서 올 수 있는 경로가 1개뿐이라 경로를 나누어서 처리했다.

### 결과
---
25,104KB	/ 216ms


### 마치며
---
처음에는 단순히 그리디 문제로 생각했지만, 중간선택이 결과에 영향을 주어 항상 최적해를 보장하지 않는다는 걸 알았다. 그래서 각 칸까지의 최대 값을 누적하는 dp로 문제를 풀었다.

dp를 2차원 배열로 풀었지만, 1차원 배열로 문제를 푼다면 최적화 가능할 거 같다.
