# BOJ 9019 DSLR
### [문제 바로가기](https://www.acmicpc.net/problem/9019)
| 난이도 | 정답률 | 분류 |
| ------ | ------ | ------ |
|  Gold IV | 21.731% | bfs |


### 구현
---
초기 숫자 `A`에서 목표 숫자 `B`로 변환하기 위한 **최소한의 명령어 나열**을 찾는 문제다.

* **네 가지 명령어 처리:**
* **D**: (2n(mod 10000))
* **S**: n-1 처리 (단, n=0 이면 9999)
* **L**: 왼쪽 자릿수 이동. 
* **R**: 오른쪽 자릿수 이동. 

### 결과
---
18,108KB	/ 196ms


### 마치며
---
처음에는 단순 시뮬레이션 문제로 생각했지만, 최소한의 명령어 나열을 구해야 한다는 점에서 BFS 문제라는걸 알았다. 

현재 풀이에서는 각 숫자마다 String 배열에 전체 경로를 저장해 문자열 연산이 반복될수록 메모리 사용량이 늘어나고 속도가 느리다.이를 최적화 하기 위해 경로 역추적 방식을 사용해 각 숫자마다 '이전 숫자'와 '사용한 명령어'만 저장해주고, 목표값에 도달 할때 역으로 거슬러 올라가면 효율을 높일 수 있을 것 같다.
